# 请用 MATLAB 写一个脚本，实现以下功能

## 输入IN_num路IQ信号和信道参数
1. 用户可选择本地文件夹中的mat文件；
2. 浮点：读取 mat 文件中的矩阵变量H（含多组信道）；读取 mat 文件中的IQ数据流变量 iq_sin_float 以及 iq_ofdm_float；
3. 定点：从txt文件中读取定点化IQ数据流，从irc，ird文件中读取信道系数和时延系数数据；irc，ird 中包含了多组（IN_num × OUT_num）信道；每组信道有T_num个TDL的tap；
4. 从txt文件读取定点IQ数据时，需按16进制读取，并按高16位为Q（虚部）、低16位为I（实部）的方式解析为有符号的int16。
5. 定点信道系数需从 .irc 和 .ird 中解析，注意它们是 32-bit 16进制字符串文本，需依据生成规则还原为int16的实虚部和延迟值。参考ChannelFixedPointTools.m。
6. 为了与 ChannelFixedPointTools.m 保持一致，建议在文档中指明：“第 m 个输入和第 n 个输出之间的信道索引映射关系为：ch = (m - 1) * OUT_num + n。

在FIR计算中，需根据 taps 中的时延值（FPGA时钟周期计数）对输入数据流进行相应的样值延迟对齐后再与系数相乘。



4. CIR update rate 和 fs 不同，需要匹配。匹配方案是在信道系数中插入重复样值，使他们在单位时间内样值数量一致。

采用零阶保持（Zero-Order Hold）策略，即在 $f_s / CIR\_update\_rate$ 个采样周期内，FIR 滤波器的信道系数保持当前快照（Snapshot）不变，直到下一个 CIR 更新点再切换下一组系数。



## FIR矩阵及FIR卷积处理单元
4. 检查数据维度是否匹配FIR矩阵的卷积操作，检查数据流数量 以及信道数量是否匹配（IN_num应和输入数据流数量一致）。OUT_num应和输出数据流数量一致。如不匹配需要扩充 数据流数量（复制多组数据流）；

5. FIR矩阵处理流程如下
* H_ch 是一个由（IN_num × OUT_num）个FIR卷积处理单元组成的 FIR矩阵，有IN_num行，OUT_num列;
* 每个FIR单元含T_num个taps（包含以FPGA时钟计数的时延、系数实部和虚部）。在计算时，输入IQ数据需先根据该tap的时延值进行数据延时（移位），再与对应系数进行复数乘法。

* 每组输出数据流由本组各列上IN_num个FIR滤波输出相加获得。

* 表示IN_num个复数IQ输入数据流，OUT_num个复数IQ输出数据流，数据流的采样率是fs;
* 每个FIR单元 含T_num个taps（含时延、系数实部和虚部），这些taps是时变的，刷新率为CIR_update_rate;

* 针对 CIR update rate 和 fs 不同，需采用“零阶保持”逻辑：在 $f_s / CIR\_update\_rate$ 的时间窗口内，保持使用同一组信道快照进行滤波。

* 每组输入数据流完成本行的所有OUT_num个FIR卷积处理单元计算，共完成IN_num组；
* 每组输出数据流由本组上各列上IN_num个FIR滤波输出相加获得；
* 对于定点计算，IN_num个FIR滤波输出相加  因为输出数据为16位，在需要具备动态截位的功能，
* 对于定点计算，乘法和加法保持 32 位全精度。为了输出 16 位 IQ 数据，需实现动态截位（寻找最佳右移位数 Shift Right），自动截位需通过比对 32位全精度结果与浮点结果，找到使量化误差最小的右移配置，并支持饱和处理。


6. 确保采样率和CIR update rate匹配的前提下分别实现所有（IN_num × OUT_num）组的浮点和定点卷积操作。浮点运算建议使用MATLAB自带的卷积函数。
7. 所有定点计算时的乘法和加法都应保持全精度，直到最后一步需要输出16bit输出IQ时做截位操作，自动截位操作需保障最小的定点化误差，可选手动截位操作。

全精度计算结果为32位（16位IQ × 16位系数），自动截位需遍历寻找一个最佳的右移位数（如 >> N），使得截位并饱和到16位后的定点输出与浮点输出的均方误差（MSE）或最大绝对误差最小，并显示该最佳移位值。

## 输出OUT_num路IQ信号
1. 分别完成浮点和定点计算获得的IQ输出信号，需要计算浮点和定点滤波处理结果的平均误差值。
2. 计算获得的OUT_num路IQ信号结果存盘为 *.mat文件和 *.txt 文件；
