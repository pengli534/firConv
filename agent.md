# MATLAB 信道卷积验证与定点分析工具需求

## 1. 目标

开发一个 MATLAB 工具，形式为“主函数 + 若干本地子函数”，用于以下目的：

1. 验证信道仿真器中 FPGA 数字逻辑的正确性。
2. 为 FPGA 中 FIR 滤波与 MIMO 计算提供定点化分析依据。
3. 支持动态截位搜索，并评估定点输出相对浮点参考输出的量化误差。

工具需要同时支持两条计算链路：

1. 浮点链路：浮点 IQ + 浮点信道 `H`
2. 定点链路：定点 IQ `.txt` + 定点信道 `.irc/.ird`

核心原则：

1. 所有浮点 IQ 数据只从 `.mat` 文件读取
2. 所有定点 IQ 数据只从 `.txt` 文件读取
3. 所有定点信道数据只从 `.irc/.ird` 读取
4. 定点链路不能直接读取 `.mat` 中的定点变量作为计算输入，因为需要同时验证 `.txt`、`.irc`、`.ird` 文件本身的正确性


## 2. 当前目录结构

工具运行时不再依赖 `ref\` 目录，当前输入数据按功能分为两个目录：

1. `signalGen\`
2. `channelGen\`

其中：

1. `signalGen\` 保存 IQ 测试信号
2. `channelGen\` 保存信道数据及定点化输出


## 3. 输入数据来源

### 3.1 IQ 数据目录

IQ 数据固定来自 `signalGen\`，当前目录中包含：

1. `fpga_test_signals.mat`
2. `sim_sine_iq.txt`
3. `sim_ofdm_iq.txt`
4. `sim_debug_real_iq.txt`
5. `sim_debug_imag_iq.txt`

### 3.2 信道数据目录

信道数据固定来自 `channelGen\`，当前目录中包含：

1. 浮点信道 `.mat`
2. 定点信道 `.irc`
3. 定点信道 `.ird`
4. 辅助检查文件，如 `_irc_decimal.txt`、`_ird_decimal.txt`
5. 参考脚本 `ChannelFixedPointTools.m`

说明：

1. 实际计算时不需要依赖 `ChannelFixedPointTools.m` 运行
2. 但 `.irc/.ird` 的解析规则必须与该脚本保持一致


## 4. 数据格式约定

### 4.1 位宽

1. IQ 数据 I/Q 各 16 bit
2. 信道系数实部/虚部各 16 bit

### 4.2 浮点 IQ `.mat`

浮点 IQ 固定从 `signalGen\fpga_test_signals.mat` 读取以下变量：

1. `fs`
2. `iq_sine_float`
3. `iq_ofdm_float`

说明：

1. 浮点验证使用 `iq_sine_float` 和 `iq_ofdm_float`
2. 调试序列不从 `.mat` 中读取
3. 调试序列仅从对应的 `.txt` 文件读取，便于 FPGA 调试时与板端数据保持一致

### 4.3 定点 IQ `.txt`

定点 IQ 文本固定从 `signalGen\` 下读取，包含：

1. `sim_sine_iq.txt`
2. `sim_ofdm_iq.txt`
3. `sim_debug_real_iq.txt`
4. `sim_debug_imag_iq.txt`

格式约定如下：

1. 每个样点对应一个 32-bit 十六进制字
2. 高 16 bit 为 Q，即虚部
3. 低 16 bit 为 I，即实部
4. 解析后恢复为有符号 `int16`

补充约定：

1. 定点 IQ `.txt` 是所有定点 IQ 运算数据的唯一来源
2. 包括 sine、OFDM、debug real、debug imag 在内的所有定点 IQ，都必须从对应 `.txt` 读取

### 4.4 浮点信道 `.mat`

从 `channelGen\` 下的信道 `.mat` 文件中读取：

1. `H`
2. `cir_up_rate`
3. `IN_num`
4. `OUT_num`

变量 `H` 的维度组织固定为：

`[Nsamples, T_num*3, IN_num*OUT_num]`

其中每个 tap 三元组的顺序为：

1. `delay`
2. `real`
3. `imag`

即：

1. `H(:, 1:3:end, :)` 为 delay
2. `H(:, 2:3:end, :)` 为 real
3. `H(:, 3:3:end, :)` 为 imag

补充约定：

1. `H` 中 delay 的原始单位是 `ns`
2. `Nsamples` 表示 CIR snapshot 数
3. `T_num` 表示每个 channel 的 tap 数

### 4.5 定点信道 `.irc/.ird`

定点信道需从 `channelGen\` 下的 `.irc` 和 `.ird` 文本恢复。

规则如下：

1. `.irc` 用于恢复定点系数实部和虚部
2. `.ird` 用于恢复时延信息
3. `.ird` 解析得到的是相邻 snapshot 的时延增量，不是绝对时延
4. 绝对时延恢复规则完全沿用 `channelGen\ChannelFixedPointTools.m`


## 5. 文件配对规则

### 5.1 IQ 信号集合

`signalGen\` 中当前包含 4 类 IQ 数据资源：

1. `iq_sine_float` 对应 `sim_sine_iq.txt`
2. `iq_ofdm_float` 对应 `sim_ofdm_iq.txt`
3. 调试实部递增序列对应 `sim_debug_real_iq.txt`
4. 调试虚部递增序列对应 `sim_debug_imag_iq.txt`

其中正式参与 FIR 浮点/定点对比验证的信号为：

1. `iq_sine_float` / `sim_sine_iq.txt`
2. `iq_ofdm_float` / `sim_ofdm_iq.txt`

调试序列：

1. `sim_debug_real_iq.txt`
2. `sim_debug_imag_iq.txt`

默认不纳入正式 FIR 结果验收。

工具需要提供一个开关，用于控制是否将调试序列加入验证流程。

### 5.2 信道集合

信道集合固定来自 `channelGen\` 中所有参与验证的信道文件组，每组由以下文件组成：

1. `<prefix>_fixedpoint.mat` 或其他与之等价的信道 `.mat`
2. `<prefix>.irc`
3. `<prefix>.ird`

兼容规则：

1. 需要兼容当前目录中 `*_fixedpoint.mat` 对应同名前缀 `.irc/.ird` 的命名方式
2. 例如 `cir_class_1tap_20ns_fixedpoint.mat` 需要与 `cir_class_1tap_20ns.irc`、`cir_class_1tap_20ns.ird` 自动配对

### 5.3 验证配对方式

验证时采用如下策略：

1. 对正式参与验证的 IQ 信号逐一处理
2. 与 `channelGen\` 中所有合法信道组逐一匹配
3. 每个正式 IQ 信号都要分别与每组信道完成浮点与定点验证
4. 当“包含调试序列”开关打开时，额外对 `sim_debug_real_iq.txt` 和 `sim_debug_imag_iq.txt` 执行同样流程

说明：

1. 浮点计算使用浮点 IQ 和浮点信道 `.mat`
2. 定点计算使用定点 IQ `.txt` 和定点信道 `.irc/.ird`
3. 浮点与定点信道文件之间采用同名前缀配对，并兼容 `*_fixedpoint.mat` 命名
4. 调试序列默认不进入正式验证流程
5. 当调试序列被纳入验证时，其输入只从 `.txt` 读取，不从 `.mat` 读取
6. 调试序列的参考计算基于 `.txt` 解析后的 `int16` 数据直接转换为 `double`，保持整数幅值


## 6. 参数与默认值

### 6.1 采样率

1. `fs` 从 `signalGen\fpga_test_signals.mat` 的变量 `fs` 读取
2. `cir_up_rate` 从信道 `.mat` 的变量 `cir_up_rate` 读取

### 6.2 FPGA 时钟频率

默认约定：

1. FPGA 时钟频率默认等于采样率
2. 默认值为 `245.76e6 Hz`
3. 本阶段程序只考虑 `clk = fs`
4. 当前阶段不处理 `clk != fs` 的情况
5. 调试时可保留相关参数接口，但本阶段计算逻辑按 `clk = fs` 实现

`clk` 与 `fs` 的关系定义如下：

1. `fs` 表示 IQ 数据流的采样率
2. `clk` 表示 FPGA 处理时钟频率
3. 本阶段固定 `clk = fs`
4. 因此在本阶段，1 个时钟计数对应 1 个 IQ 采样间隔

时延换算建议按以下顺序处理：

1. 从 `H` 中读取 delay，单位为 `ns`
2. 换算为时钟计数：`delay_clk = round(delay_ns * clk * 1e-9)`
3. 在本阶段，由于 `clk = fs`，样值延迟直接取 `delay_clk`

补充约定：

1. 本阶段样值延迟直接取 `delay_clk`
2. `clk != fs` 的情况不属于本阶段范围
3. 下一阶段再扩展支持 `2*clk = fs`
4. 到下一阶段时，需要考虑双路并行处理，所有信号路径和信道路径均拆分为奇偶两路
5. 若未来遇到 `clk != fs` 但仅为轻微差异的情况，则样值移位按 `round(delay_time * fs)` 计算，其中 `delay_time = delay_clk / clk`

### 6.3 IN/OUT 规模

从信道 `.mat` 中读取：

1. `IN_num`
2. `OUT_num`

可得总信道channel数为：

`IN_num * OUT_num`


## 7. 前置检查

在开始计算前必须检查：

1. `signalGen\fpga_test_signals.mat` 是否存在
2. 浮点 IQ 所需变量是否存在
3. 定点 IQ `.txt` 文件是否存在
4. 信道 `.mat` 是否包含 `H`
5. 信道 `.mat` 是否包含 `cir_up_rate`、`IN_num`、`OUT_num`
6. `.irc`、`.ird` 是否存在且能与 `.mat` 成功配对
7. `H` 的维度是否满足 `[Nsamples, T_num*3, IN_num*OUT_num]`

若文件中不存在 `H`，需给出友好提示。


## 8. 通道索引映射

为与 `ChannelFixedPointTools.m` 保持一致，信道索引定义为：

`ch = (m - 1) * OUT_num + n`

其中：

1. `m` 为输入通道编号，范围 `1 ~ IN_num`
2. `n` 为输出通道编号，范围 `1 ~ OUT_num`


## 9. FIR 与 MIMO 计算规则

### 9.1 FIR 矩阵结构

整个信道矩阵由 `IN_num × OUT_num` 个 FIR 单元组成：

1. 行对应输入通道
2. 列对应输出通道

每个 FIR 单元包含 `T_num` 个 taps，每个 tap 包含：

1. 时延
2. 系数实部
3. 系数虚部

### 9.2 浮点参考计算

浮点参考结果定义为：

1. 使用浮点 IQ
2. 使用浮点信道 `H`
3. 执行理想复数 FIR 卷积

### 9.3 时延处理

在 FIR 计算中，需根据每个 tap 的时延值对输入样值进行延迟对齐，再与对应复系数相乘。

实现链路定义为：

1. 从 `H` 中读取 delay，单位为 `ns`
2. 使用 FPGA 时钟频率将其转换为时钟计数
3. 在本阶段，由于 `clk = fs`，时钟计数直接作为样值移位
4. 根据样值移位执行延时对齐

### 9.4 输出求和规则

每一路输出由对应输出列上的 `IN_num` 路 FIR 结果求和得到。

即：

1. 每个输入通道分别经过到各输出通道的 FIR
2. 同一输出通道上对所有输入通道结果求和


## 10. CIR update rate 与采样率匹配

当 `fs` 与 `cir_up_rate` 不同时，采用零阶保持策略。

规则如下：

1. 在 `fs / cir_up_rate` 对应的采样窗口内保持当前 snapshot 不变
2. 到达下一个更新点时切换到下一组 snapshot
3. 若 `fs / cir_up_rate` 不是整数，则采用 `round` 方式取整


## 11. 输入路数容错

若 IQ 输入路数与 `IN_num` 不一致，则允许为了调试和验收进行容错扩展：

1. 按已有输入数据流循环复制
2. 直到路数达到 `IN_num`

说明：

1. 这是调试/验收阶段的容错逻辑
2. 后续正式版本预计会直接生成多通道 IQ 数据


## 12. 定点计算与动态截位

### 12.1 定点运算规则

1. 单次 16 bit IQ 与 16 bit 系数相乘时，单个乘积结果按 32 bit 全精度保存
2. 复数乘法内部的实部与虚部中间结果，也应按不低于 32 bit 的精度保存
3. 多个 tap 的卷积累加，以及多个输入通道的求和，其累加精度可能高于 32 bit
4. 因此“32 bit 全精度”只适用于单次乘法结果，不应限制最终累加器位宽
5. 实现时应保证累加过程中不因位宽不足产生额外截断
6. 所有累加完成后，才允许执行最终输出截位到 16 bit
7. 截位后需进行饱和处理

建议实现原则：

1. 乘法结果至少保留 32 bit
2. 累加结果使用扩展位宽保存，位宽可根据 `T_num` 和 `IN_num` 估算，或直接使用 MATLAB 整型/`fi` 的更高精度配置
3. 自动截位搜索应基于最终累加后的全精度结果，而不是基于中途被截断后的结果

### 12.2 自动截位

自动截位需搜索最佳右移位数 `Shift Right = N`，要求如下：

1. I/Q 共用同一个 shift
2. 正式搜索范围为 `0 ~ 18`
3. 截位并饱和到 16 bit 后，与浮点参考输出比较
4. 默认优化目标为归一化后的 `MSE` 最小
5. 除正式搜索结果外，还需要额外输出“宽搜索建议值”，用于提示更宽搜索范围下的推荐 shift

可附加输出：

1. `max abs error`

### 12.3 手动截位

工具需支持手动指定右移位数，用于调试和对比验证。


## 13. 误差定义

误差评估规则如下：

1. 主指标为归一化后的 `MSE`
2. 误差容忍度目标为 `< 1e-4`
3. 比较尺度为归一化到满量程 `[-1, 1]`
4. 复数误差按 `abs(e)` 理解
5. 浮点与定点结果比较时保留完整卷积长度

调试序列的特殊规则：

1. 调试序列的目标更偏向 FPGA 原始码流核对
2. 因此调试序列误差比较时不做归一化
3. 调试序列参考输入由 `.txt` 解析出的 `int16` 直接转换为 `double`，保持整数幅值
4. 正式信号 `sine` 和 `OFDM` 仍按归一化口径统计误差

输出中需要至少包含：

1. 每个测试项的误差值
2. 每个测试项的最佳截位值
3. 每个测试项的“宽搜索建议值”


## 14. 输出结果

工具需要输出 `OUT_num` 路 IQ 信号，并保存为：

1. `*.mat`
2. `*.txt`

其中 `.txt` 输出格式需与 `signalGen\` 中 IQ 文本格式一致：

1. 32-bit 十六进制
2. 高 16 bit 为 Q
3. 低 16 bit 为 I

输出结果同时需要：

1. 打印到命令行
2. 保持格式清晰易读

结果输出目录固定为：

`output\`

输出 `.mat` 的变量命名规范由实现阶段统一定义即可，不再额外指定。


## 15. 批量验收规则

批量验收时：

1. 扫描 `channelGen\` 下所有含 `H` 的信道 `.mat`
2. 与对应同名前缀 `.irc`、`.ird` 配对
3. 对正式参与验证的 IQ 信号逐个完成验证
4. 若开关打开，则额外对两路调试序列执行验证
5. 单个文件失败时记录错误并继续下一个
6. 误差报告按文件逐项打印，不要求汇总总表


## 16. 异常处理与兼容性

代码要求如下：

1. 包含必要注释
2. 包含异常处理
3. 兼容 MATLAB R2021a 及以上
4. 仅依赖 MATLAB 自带能力和 `fi` 工具箱
5. 不依赖第三方工具箱


## 17. 验收标准

需要满足以下验收目标：

1. 能正确识别 `signalGen\` 和 `channelGen\` 中的输入文件
2. 能正确识别变量 `H`
3. 能正确识别 IQ 输入资源，并区分正式验证信号与调试信号
4. 能通过开关控制是否将调试序列加入验证
5. 能完成浮点链路与定点链路计算

7. 单个信道文件验证失败时不会中断整个批量过程
8. 能显示自动或手动截位配置值
9. 能额外输出“宽搜索建议值”
10. 量化误差满足归一化后 `< 1e-4`

4. 所有浮点 IQ 只从 `.mat` 读取；所有定点 IQ 只从 `.txt` 读取；所有定点信道只从 `.irc/.ird` 读取。

6. 信道文件配对需兼容 `*_fixedpoint.mat` 与同前缀 `.irc/.ird` 的命名方式。
7. 结果输出目录固定为 `output\`。
8. 自动截位正式搜索范围调整为 `0 ~ 18`。
